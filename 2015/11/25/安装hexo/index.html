安装hexo

发表于 2015-11-25   |   分类于 hexo   |  

花了两天的时间终于搞定了hexo + gitcafe的博客的搭建。在我现在初次使用看来，它是多么的适用于我们程序员，所谓的逼格无比之高。那么接下来就开一个小小的教程，去搭建一个逼格如此之高的博客。参考来自网络上的各位大神，我在此借用一下各位的研究，希望不要介意。

安装前准备

github/gitcafe

在我刚开始搭建的时候也是使用的github，但由于我的主要使用地为国内，还没有那么大的能力发展到国外，所以就是用了速度在国内相对快一点的gitcafe（手动靠ping来测试的，不专业，可以忽略），基本配置没什么区别。所以首先你得有一个github账号，或者gitcafe，接下来我会主要介绍gitcafe的安装。

访问：gitcafe.com

相信注册就不用多说了，注册好登入自己的账号，新建一个项目

注意：项目名称为你的用户名，项目主页为 http://你的用户名.gitcafe.io

然后就是安装git bash了，如果你不知道git bash是什么，那还请在研究研究github的使用。
当然，你也可以跟着我继续往下走，基本的使用方法我也会介绍的。我目前也没有git bash特别的下载地址，我就随便推荐一个，这个网上一搜一大片。git bash下载地址

git bash的简单使用和ssh

参考自：GitCafe 官方帮助文档

右键点击任意地方（桌面或文件夹或文件），选择Git Bash Here，会弹出一个命令行工具

  1. 进入ssh目录

    如果没有ssh目录可以手动创建一个。mkdir ~/.ssh。但由于权限问题可能会创建失败，不过没关系，失败了就忽略这一步。

  2. 生成ssh密钥

    将其中的YOUR_EMAIL@YOUREMAIL.COM替换成你的email地址。接下来会提示你设置passphrase 口令，如果你需要，就设置一下，要输入两次。如果不需要，就果断一通回车。

  3. 在刚才的操作下已经生成了ssh密钥了，屏幕上会提示你密钥的存放地址，进入这个地址，用编辑器打开公钥id_rsa.pub这个文件，记得是公钥。复制id_rsa.pub中的内容到剪贴板。
  4. 打开你的gitcafe，进入GitCafe –>账户设置–>SSH 公钥管理设置项，点击添加新公钥 按钮，Title可以随便设置，Key文本去将刚才你复制的内容粘贴进去就好啦。然后点击更新。就完成啦。

  5. 当然，到底有没有成功，还是很有必要测试一下的。

    输入上述命令后，由于第一次连接，可能会有警告，不过这些都不要紧，你就回车回车回车，只要出现了Hi USERNAME! You've successfully authenticated, but GitCafe does not provide shell access.,那就成功啦，要是没有，你就再好好看看上述步骤吧。
