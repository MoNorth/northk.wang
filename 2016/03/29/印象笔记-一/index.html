JS cookie

js设置cookie和获取的方法是通过document.cookie去操作的。

设置：document.cookie=name+value 设置多条cookie不会覆盖。

数量和大小限制：

  1. Firefox每个域名cookie限制为50个。
  2. Opera每个域名cookie限制为30个。
  3. Safari/WebKit貌似没有cookie限制。但是如果cookie很多，则会使header大小超过服务器的处理的限制，会导致错误发生。
  4. Firefox和Safari允许cookie多达4097个字节，包括名（name）、值（value）和等号。
  5. Opera允许cookie多达4096个字节，包括：名（name）、值（value）和等号。
  6. Internet Explorer允许cookie多达4095个字节，包括：名（name）、值（value）和等号。

<link>与@import

用法

  1. link : <link rel="stylesheet" type="text/css" href="文件地址">
  2. @import : <style type="text/css">@import url("文件地址")</style>

区别

  1. link是HTML标签，除了加载CSS外，还可以定义RSS等其他事务；@import属于CSS范畴，只能加载CSS。
  2. link引用CSS时，在页面载入时同时加载；@import需要页面网页完全载入以后加载。
  3. link是HTML标签，无兼容问题；@import是在CSS2.1提出的，低版本的浏览器不支持。
  4. link支持使用Javascript控制DOM去改变样式；而@import不支持。

relative与translate

作用

从布局的角度来看，两个的效果是一样的，都是根据自身位置进行偏移。

区别

  1. translate可以简单地作用于position:absolute的元素上面，而position:relative还得加额外的标签。
  2. 使用translate时，可以让GPU参与运算，动画的 FPS更高。使用position时，最小的动画变化的单位是1px，而使用translate参与时，可以做到更小（动画效果更加平滑）
  3. 总的来说，position主要用于布局，而transform主要是动画。
  4. 参考资料 http://paulirish.com/2012/why-moving-elements-with-translate-is-better-than-posabs-topleft/

localStorage和sessionStorage

localStorage sessionStorage与cookie

  1. cookie数据始终在同源的http请求中携带（即使不需要），即cookie在浏览器和服务器间来回传递。而sessionStorage和localStorage不会自动把数据发给服务器，仅在本地保存。
  2. cookie数据还有路径（path）的概念，可以限制cookie只属于某个路径下。
  3. cookie数据基本不能超过4k(具体看上文)，同时因为每次http请求都会携带cookie，所以cookie只适合保存很小的数据，如会话标识。sessionStorage和localStorage 虽然也有存储大小的限制，但比cookie大得多，可以达到5M或更大。
  4. sessionStorage：仅在当前浏览器窗口关闭前有效，自然也就不可能持久保持；localStorage：始终有效，窗口或浏览器关闭也一直保存，因此用作持久数据；cookie只在设置的cookie过期时间之前一直有效，即使窗口或浏览器关闭。
  5. sessionStorage不在不同的浏览器窗口中共享，即使是同一个页面；localStorage 在所有同源窗口中都是共享的；cookie也是在所有同源窗口中都是共享的。

webWorker

Web Worker的基本原理就是在当前javascript的主线程中，使用Worker类加载一个javascript文件来开辟一个新的线程，起到互不阻塞执行的效果，并且提供主线程和新线程之间数据交换的接口：postMessage，onmessage。

演示程序

解析

  1. 通过 worker = new Worker( url ) 加载一个JS文件来创建一个worker，同时返回一个worker实例。
  2. 通过worker.postMessage( data ) 方法来向worker发送数据。
  3. 绑定worker.onmessage方法来接收worker发送过来的数据。
  4. 可以使用 worker.terminate() 来终止一个worker的执行。
  5. 子线程可以通过postMessage( data ) 方法来向主线程发送数据。
  6. 子线程可以绑定onmessage方法来接收主线程发送过来的数据。

以上示例与资料来源于 http://blog.jobbole.com/30592/

webSocket

WebSocket API是下一代客户端-服务器的异步通信方法。该通信取代了单个的TCP套接字，使用ws或wss协议，可用于任意的客户端和服务器程序。WebSocket目前由W3C进行标准化。WebSocket已经受到Firefox 4、Chrome 4、Opera 10.70以及Safari 5等浏览器的支持。

WebSocket API最伟大之处在于服务器和客户端可以在给定的时间范围内的任意时刻，相互推送信息。WebSocket并不限于以Ajax(或XHR)方式通信，因为Ajax技术需要客户端发起请求，而WebSocket服务器和客户端可以彼此相互推送信息；XHR受到域的限制，而WebSocket允许跨域通信。

Ajax技术很聪明的一点是没有设计要使用的方式。WebSocket为指定目标创建，用于双向推送消息。
