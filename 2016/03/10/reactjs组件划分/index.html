Reactjs组件的划分

学习React也已经有一段时间，总感觉是野路子多点，毕竟React用起来还是塑造性相当强的，像我这种没有阅读过专业书籍，也没有看过“别人家”代码的小白，只是个名副其实的野路子吧。不过用起来确实很溜，比起之前用的SUI等框架而言，React简直就是神器，它的虚拟DOM使得页面变得十分顺畅。这块就不应该将她称之为框架了，更准确的说是前端的构建工具了。总之用起React就一句话，妈妈再也不用担心我的单页面大数据了！

浅谈React组件化编程

首先引用 官网 上的一句话：

React都是关于构建可复用的组件。事实上，通过React你唯一要做的事情就是构建组件。得益于其良好的封装性，组件使代码复用、测试和关注分离（separation of concerns）更加简单。

在我的实际使用中更加深入的理解了这句话的含义。React就是组件化的编程，越来越像客户端了。组件可以复用，可以嵌套，在实际生产中这样做也大大节省可代码量，也是你的代码看上去更加整洁。可是，上述的一切都是在你的组件划分还算ok的基础上说的。如果组件划分的一塌糊涂，就别提怎么复用嵌套了。腾讯的面试官曾经问我React时问道：你的React项目组件是怎么划分的。所以说，React的组件划分真的很重要。

UI设计 ：

你要做的第一件事是，为所有组件（及子组件）命名并画上线框图。假如你和设计师一起工作，也许他们已经完成了这项工作，所以赶紧去跟他们沟通！他们的 Photoshop 图层名也许最终可以直接用于你的 React 组件名。

然而你如何知道哪些才能成为组件？想象一下，当你创建一些函数或对象时，用到一些类似的技术。其中一项技术就是单一功能原则，指的是，理想状态下一个组件应该只做一件事，假如它功能逐渐变大就需要被拆分成更小的子组件。

由于你经常需要将一个JSON数据模型展示给用户，因此你需要检查这个模型结构是否正确以便你的 UI （在这里指组件结构）是否能够正确的映射到这个模型上。这是因为用户界面和数据模型在 信息构造 方面都要一致，这意味着将你可以省下很多将 UI 分割成组件的麻烦事。你需要做的仅仅只是将数据模型分隔成一小块一小块的组件，以便它们都能够表示成组件。

组件划分 ：

随便谈谈

其实人家已经讲的很清楚了，组件的划分只要把握住了单一功能原则,基本上也就没有什么问题了。下面我就简单谈一下我的看法吧。这只是我的看法，不具备权威性

  1. 从最重要并最小的单元开始划分
  2. 找出与之相关并行使相同职能并且可以一起重复使用的元素。
  3. 将其看为组一，找寻其父级元素。
  4. 查找其父级元素是否包含除组一外的其它元素。
  5. 如果不存在，就继续找寻其父级，直至有其他元素包含在其父级或已为最顶级元素。
  6. 如果包含其他元素，就以其他元素为组二。并在其他元素中找寻最重要并最小的单元从，重复以上步骤。

分析 ：

  1. 找出最重要最小的元素，无可置疑，肯定是内容了。
  2. 找出其与之相关并行使相同职能的元素。就是后面的图片
  3. 寻找其父级元素
  4. 它并没有其它的子元素了，那就继续寻找它的父级。
  5. 它的顶部一个图片轮询，先将其也化为一个组
  6. 现在将其看作整体，查看它内部最重要最小的元素，就是它本身，那么就不用继续了。接着找绿色组的父级
  7. 它的顶部和底部有标题栏和导航栏。将其分别化成组。
  8. 接下来就是以这两个组为整体，分别继续划分。我这里就不多化了。
